1.2 豆包+即梦+SD — 动图制作全流程 文档01

核心工具链：豆包（文案）→ 即梦（图像）→ SD/ComfyUI（产品融合）→ 即梦视频（动图）

完整工作流

📝 第一步：文案与背景图生成（豆包）
  • 告知豆包这是一个成品设计/手机海报文案
  • 背景图可复用已有或由豆包新生成
  • 抽卡思维：豆包有一定随机性，需多次生成筛选最优结果
  • 场景配图同样可由豆包生成
🖼️ 第二步：详情页视觉生成（即梦）
参数设置值
模型3.1 版本
标签1K
尺寸512 × 2016（偏长条形，适配手机海报/详情页比例）

即梦作用：提供灵感和基础框架，最终仍需手动精调。

🎨 第三步：在线SD溶图 —— 替换产品
  1. PS预处理：用Photoshop打开图片，裁剪不需要的部分，用移除工具移除原有产品
  2. 放置新产品：把自己的产品图片放入合适位置
  3. 保存：Ctrl+Shift+Alt+S 保存
  4. 在线SD溶图：打开在线溶图工具，点击"在ComfyUI中运行"，上传处理好的图片，系统自动做图像融合处理
  5. 保存融合结果
🎬 第四步：即梦视频生成 —— 制作动图
  1. 打开即梦 → 灵感 → 视频生成
  2. 上传融合好的产品图
  3. 输入关键词（提示词）
  4. 选择快速模式生成
  5. 查看生成的动图效果
⚠️ 关键认知：AI工具输出有随机性（抽卡思维），需多次生成筛选最优结果。手动调整不可避免——配图位置、图标数量、产品替换等环节都需要人工介入。

1.3 即梦+SD智能体 — 全链路详情页制作 文档02

最大特色：两个AI智能体的分工协作——智能体①负责详情页策划，智能体②负责提示词生成。

核心工作流全景

📋 产品需求 → 🔍 竞品分析 [智能体1] → 📐 策划架构 [智能体1] → ✏️ 生成提示词 [智能体2] → 🎨 即梦出图 → 🔧 PS替换+龙图融合 → 🎬 图生视频 → ✅ 成品交付

四大板块详解

① 需求策划与智能体

  • 智能体1：详情页需求文档——分析竞品、策划文案
  • 智能体2：详情页提示词需求——自动编写AI绘画提示词
  • 上传3张竞品图片用于分析
  • 按"每一屏"输出详情页结构
  • 输出到Excel表格管理，与客户确认

② 提示词与即梦出图

  • 将策划表需求复制给智能体2
  • 风格设定：上传参考图 + "简约风、米白色调"
  • 通过豆包获取专业产品外观描述
  • 即梦设置：模型3.1、标清1K、512×2016
  • 提示词格式："产品名+详情页设计" + 画面描述

③ 后期处理与动图

  • PS内容识别填充移除原产品
  • 钢笔工具或对象选择工具抠图
  • 龙图工作流：自动光影融合+投影补齐
  • 即梦图生视频：豆包辅助写提示词

④ 自定义AI智能体创建

  • 入口：AI平台→更多→智能体→创建
  • 基本设置：名称、设定描述（系统提示词）、权限（私密）
  • 系统提示词框架：角色定位、工作流程、技能说明、总体规则
  • 创建完成后需要审核才能使用
📋 智能体系统提示词框架（模板） 
角色：你是一个详情页的策划师
技能：分析竞品、策划文案、输出详情页架构
工作流程：
第一步：让用户上传详情页图片（三张竞品）
第二步：上传完成后，进入文案策划环节，用户上传产品信息
第三步：结合产品及竞品分析，写出对应文案
第四步：输出详情页架构
规则：按步骤一步一步执行

2.2 即梦AI+智能体猫粮详情页 文档05

独特价值：完全不需要人工撰写文案，零文案输入，全由AI生成。

使用工具

工具用途
豆包产品外观描述、创建详情页智能体、生成详情页文案
即梦AI详情页图片生成、产品动图制作
set（聊天式AI工具）发送竞品参考，辅助生成文案
PS + 在线溶图产品替换与图像融合

操作流程

  1. 竞品参考：发送竞品参考给set，让AI理解行业风格和文案调性
  2. 产品描述：将产品图片发送给豆包，使用提示词「从设计师的角度描述产品外观」
  3. 生成文案：复制到表格，检查修改备用
  4. 创建智能体：豆包平台创建 → 填写智能体描述文本 → 权限私密 → 发送需求文案
  5. 即梦出图：模型3.1、512×2016、分屏每次2屏
  6. 产品替换与动图制作

核心方法论

方法论说明
竞品参考法先给AI看竞品，让AI理解品类风格和文案调性
双工具联动豆包负责文案与智能体管理，即梦负责视觉生成
分屏出图法每次只生成2屏，便于精细化控制和修改
提示词迭代法先出初版，检查不足后修改提示词再刷新

详细工作流

阶段工具核心动作

独特技巧

📐 俯视视角生成

方案一：让豆包AI生成俯视视角的产品描述词，用即梦AI生成

方案二：将产品图片上传给即梦AI，选择智能处理功能，让AI将产品转换成俯视视角

🎞️ 首尾帧动画

  1. 让AI生成一张只有水面的空白版图片（无产品无文字）
  2. 保持尺寸一致
  3. 即梦AI上传首帧（完整图）和尾帧（空白背景图）
  4. AI自动生成平滑过渡动画
⚠️ 关键要点：AI描述词的精确度决定效果——产品颜色、材质、视角等细节描述越精准越好。双引号去除、叶子遮挡关系、投影融合等细节处理决定专业度。

提示词迭代优化

  • 首版生成：AI生成的产品与实际有差距（如金色麦克风 vs 实物米白色）→ 修改关键词

  • 第二轮：修改后匹配度显著提升
抠图技巧：推荐使用钢笔工具而非快速选择工具，可获得更精确的选区。注意调整容差参数到较小值。

六步法详解

步骤内容工具/手段
第一步分析需求解读白底图产品、文案需求、风格参考
第二步DeepSeek生成提示词模板填入产品信息+外观+分屏规划

PS光效处理技巧

  1. 新建图层，用画笔工具选蓝色，点一个大光点
  2. Ctrl+T → 右键 → 透视变形，调整光的角度
  3. 将光放到镜头位置
  4. 加图层蒙版，用画笔过渡

4.3 豆包+稿定AI+ChatGPT毛球修剪器 文档17

新增核心内容：复杂产品多角度拍摄 + ChatGPT image2产品精修 + 逐屏优化策略。

新增核心技术

📸 复杂产品多角度拍摄

结构不对称或细节多的产品，需拍摄多角度照片（底部+顶部）供AI参考，避免AI生成错误的结构理解。

✨ AI精修产品图（ChatGPT image2）

1. 严格还原上传的产品图
2. 将产品处理为商业级电商精修图
3. 产品完美无瑕，消除所有瑕疵杂质
4. [细节模块 - 根据产品修改]

🎯 逐屏优化策略

一次性批量生成15屏后，对不满意的屏进行逐屏单独生成优化。推荐做法：第1屏只需正面视角 → 只上传正面 + 修改对应提示词 → 单独生成。

6.3 Codex+LibTV护肤水乳套装 文档20

核心创新：Codex的Skill（技能）机制——将整套详情页制作流程封装成可复用的AI智能体模块，实现全自动化闭环。

核心概念：Skill（技能）机制

📦 Skill 的本质

将完整的详情页策划与提示词编写流程封装成可复用的AI智能体模块。每次做不同详情页时只需调用这个技能，无需重复发送大量提示词。

🎯 Skill 1：详情页策划技能

接收竞品和产品信息，输出详情页每屏文案和架构。定位：电商详情页策划师。流程：①分析竞品 → ②接收产品 → ③输出分屏文案

✏️ Skill 2：提示词编写技能

接收策划表格，输出每屏对应的AI生图提示词。定位：资深设计师及AI生图提示词工程师

⚙️ Skill 创建方法

直接通过对话让Codex根据需求写出。给AI设定角色定位，描述所需技能和执行流程，AI自动生成JSON/Skill格式的配置。

关键经验

1 直接让AI生图效果不佳——仍需要"先策划、再写提示词、最后出图"的分步流程
2 Skill是复用核心——将流程封装成技能可大幅提升效率
3 提示词也可由AI自动生成——智能体提示词本身也可通过对话获得
4 权限安全需谨慎——不建议给予Codex完全操控电脑的权限
5 替代方案——如果无法使用Codex，可用豆包完成类似效果

🎯 核心操作技巧TOP10

  1. 分屏生成：每次生成2~3屏，便于逐屏校对和精控
  2. 色调统一：在提示词中预设整体色调，保持跨屏一致性
  3. 逐屏生成优于批量：单屏优化可上传对应角度产品图 + 定制提示词
  4. 参考图控制：提示词中标注"参考图X的特效/排版"实现精准风格引导
  5. 竞品一张一张上传：避免AI分析不全
  6. 逗号改空格：即梦AI中标题逗号改空格可实现自动换行
  7. 不要表格：要求AI不要表格格式，获得纯文本提示词便于复制
  8. 产品外观绑定：将产品外观描述加入提示词，使生成的产品与实物匹配
  9. 多模型切换：不同模型效果不同，对比选择最佳
